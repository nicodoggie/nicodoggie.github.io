Developing with ES6

In order to use XSplit JS Framework on your ES6-based project, you will have to use Browserify and the loaders required to parse your ES6 code. The advantages of using ES6 is that you can utilize the modular nature of XSplit JS Framework to better organize your code.

On this tutorial, we will be using Webpack to bundle your code, and at the same time, transpile your ES6 code to ES5.

Using Webpack

Please head to their website to get a glimps on how to configure webpack for your use. We will list the loaders that we would need to get our project running.

NPM Modules needed

  • XJS Framework
    npm install --save-dev xjs-framework
  • Webpack
    npm install --save-dev webpack

Required loaders

  • Babel-loader
    npm install --save-dev babel-loader
  • TypeScript loader
    npm install --save-dev ts-loader

You might notice that we also need a typescript loader. This is required since the source codes of XSplit JS Framework is written on TypeScript, so we would obviously need to have a loader for webpack to parse the files.

Please do check out the documentation of webpack, babel-loader, and ts-loader on how to configure your webpack configuration file to load up both required loaders. This is necessary to transpile your ES6 code to ES5.

Using XJS on ES6

Creating XSplit plugins with ES6 as your environment would allow you to import modules that you would only need on your individual modules. To better understand what we mean, here's an example code:

import {ready, Scene} from 'xjs-framework';

ready().then(Scene.getActiveScene).then(scene => {
  return scene.getItems();
}).then(items => {
  // Do something with the items
});
