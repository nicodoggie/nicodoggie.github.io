Developing with ES2015

In order to use XSplit JS Framework on your ES2015-based project, you will have to use some kind of bundler such as Webpack or Browserify, and the loaders/transpilers required to transform your ES2015 code. The advantages of using ES2015, in addition to its improved language features, is that you can utilize the modular nature of XSplit JS Framework to better organize your code.

On this tutorial, we'll discuss on how to use both Webpack and Browserify to bundle and transpile your code to ES5.

Using Webpack

Please head to their website to get a glimpse on how to configure webpack for your use. We will list the loaders that we would need to get our project running.

NPM Modules needed

  • XJS Framework
    npm install --save-dev xjs-framework
  • Webpack
    npm install --save-dev webpack

Required loaders

  • Babel-loader
    npm install --save-dev babel-loader
  • TypeScript loader
    npm install --save-dev ts-loader

You might notice that we also need a TypeScript loader. This is required since the source code of XSplit JS Framework is written in TypeScript.

Please do check out the documentation of webpack, babel-loader, and ts-loader on how to configure your webpack configuration file to load up both required loaders.

Using XJS on ES2015 with Webpack

Creating XSplit plugins with ES2015 as your environment would allow you to import only modules that you need in your individual modules. To better understand what we mean, here's some sample code:

import {ready, Scene} from 'xjs-framework';

ready().then(Scene.getActiveScene).then(scene => {
  return scene.getItems();
}).then(items => {
  // Do something with the items
});

We have an example plugin project written in ES2015. You can actually make use of the webpack configuration of that project for your own project. Check it out here.

Using Browserify

Please head to their website to get you started with browserify. We will list the presets that we would need to get our project running.

NPM Modules needed

  • XJS Framework
    npm install --save-dev xjs-framework
  • Browserify
    npm install --save-dev browserify
  • Babelify
    npm install --save-dev babelify

Required Presets

  • ES2015 Preset
    npm install --save-dev babel-preset-es2015

Using XJS on ES2015 with Browserify

import xjs from '../node_modules/xjs-framework/dist/xjs-es2015.js';

xjs.ready().then(xjs.Scene.getActiveScene).then(scene => {
  return scene.getItems();
}).then(items => {
  // Do something with the items
});

The main difference between Browserify and the Webpack approach is that unlike Webpack, Browserify makes use of the bundled version of XJS Framework.
