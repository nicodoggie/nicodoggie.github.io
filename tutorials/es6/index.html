Developing with ES6

In order to use XSplit JS Framework on your ES6-based project, you will have to use some kind of bundler such as Webpack, and the loaders required to parse your ES6 code. The advantages of using ES6, in addition to its improved language features, is that you can utilize the modular nature of XSplit JS Framework to better organize your code.

On this tutorial, we will be using Webpack to bundle your code, and at the same time, transpile your ES6 code to ES5.

Using Webpack

Please head to their website to get a glimpse on how to configure webpack for your use. We will list the loaders that we would need to get our project running.

NPM Modules needed

  • XJS Framework
    npm install --save-dev xjs-framework
  • Webpack
    npm install --save-dev webpack

Required loaders

  • Babel-loader
    npm install --save-dev babel-loader
  • TypeScript loader
    npm install --save-dev ts-loader

You might notice that we also need a TypeScript loader. This is required since the source code of XSplit JS Framework is written in TypeScript.

Please do check out the documentation of webpack, babel-loader, and ts-loader on how to configure your webpack configuration file to load up both required loaders.

Using XJS on ES6

Creating XSplit plugins with ES6 as your environment would allow you to import only modules that you need in your individual modules. To better understand what we mean, here's some sample code:

import {ready, Scene} from 'xjs-framework';

ready().then(Scene.getActiveScene).then(scene => {
  return scene.getItems();
}).then(items => {
  // Do something with the items
});
