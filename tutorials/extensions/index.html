Developing Extension Plugins

Extensions occupy their own window separate from the main XSplit Broadcaster application. They can be used to do many kinds of tasks in the global scope of the application.

How to use extensions

Extension plugins are currently only available to XSplit premium license holders. To enable extensions in the application, go to Tools > General Settings > Advanced, and check the Enable Global Script checkbox.

While the extension plugin store is not yet available, extensions may be added through their URLs. To do this, go to Tools > Extensions > Add Extension, and input your extension's URL. It should now appear in the same Extensions menu until you choose to remove it.

What extensions can do

This section outlines some of the possible things one can do with extensions. Note that this list is not exhaustive. Explore the API reference to learn more, or contact us at our support channels for any inquiries or feature requests.

  • Adding new items to the stage. Extensions are capable of adding system-related items: CameraDevice, Game, Microphone (more will be added in the future). Simply call the relevant system call (System.getCameraDevices(), for example) and call addToScene() on the appropriate object. Note that this is currently limited to adding to the currently active scene.
  • Manipulating sources. Extensions are allowed to manipulate sources, their properties, and even their custom configuration objects just as a properties window might, as detailed here.
  • Setting application-wide properties. Extensions are much more powerful than sources and are able to manipulate the application itself. For example, they may use the App class change the transition settings when changing scenes. See the API reference for more information.
