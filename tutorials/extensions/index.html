Developing Extension Plugins

Extensions are powerful plugins that manipulate the application, its scenes, and its sources. They can be used to do many kinds of tasks to enhance your workflow or your streams.

Basics

Extensions are now part of XSplit's plugin store. There are a number of preinstalled extensions for you to try out. To add your own custom extension, simply write an HTML page, and add it through the Extensions > Add custom extension... menu.

What extensions can do

This section outlines some of the possible things one can do with extensions. Note that this list is not exhaustive. Explore the API reference to learn more, or contact us at our support channels for any inquiries or feature requests. Generally speaking, extensions have access to most of the API exposed by XSplit Broadcaster. You can also take a look at the default extensions for XSplit to gain a better idea of their potential.

  • Adding new sources to the stage. Extensions are capable of adding system-related sources through classes such as CameraDevice. Simply call the relevant call (System.getCameraDevices(), for example) and call addToScene() on the appropriate object. Please peruse the set of classes in the System module to see a list of entities that can be added to the stage as a new source. Note that this is currently limited to adding to the currently active scene.
  • Manipulating sources and video items. Extensions are allowed to manipulate sources, their properties, and even their custom configuration objects just as a source properties window might, as detailed here.
  • Setting application-wide properties. Extensions are much more powerful than sources and are able to manipulate the application itself. For example, they may use the App class change the transition settings when changing scenes. See the API reference for more information.
  • Listen to important application events.The ExtensionWindow class is an event emitter that is useful for responding to application events that extensions are able to receive. scene-load is one such event. (In addition, this class also allows you to set windowing options for extensions.)
