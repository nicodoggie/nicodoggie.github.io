Developing Extension Plugins

Extensions occupy their own window separate from the main XSplit Broadcaster application. They can be used to do many kinds of tasks in the global scope of the application.

Basic tasks

This section outlines some of the possible things one can do with extensions.

  • Adding system items. Extensions are capable of adding system-related items: CameraDevice, Game, Microphone (more will be added in the future). Simply call the relevant system call (System.getCameraDevices(), for example) and call addToScene() on the appropriate object. Note that this is currently limited to adding to the currently active scene.
  • Manipulating sources. Extensions are allowed to manipulate sources, their properties, and even their custom configuration objects just as a configuration window might, as detailed here. (Please see the Issues section below for some relevant information.)
  • Setting application-wide properties. Extensions are much more powerful than sources and are able to manipulate the application itself. For example, they may use the App class change the transition settings when changing scenes. See the API reference for more information.

Issues and possible gotchas

This section details some of the current issues related to extensions and/or the framework in general.

  • Non-active sources. When manipulating source properties, it is currently not possible to change any of their properties if the item is not in the currently active scene, or if “Keep Source in Memory” is not checked for that item.
  • Handling multiple scenes. If your extension requires manipulation of more scenes, it is imperative that the scenes be initialized first. You can do this by calling `Scene.initializeScenes()`. This function is not automatically called by the framework for memory purposes.
