Debugging Your Plugins

XSplit Broadcaster allows you to debug your pages by exposing the DevTools. To do this, create a shortcut to XSplit.Core.exe, and right-click to open the properties window to add the startup parameter: remotedebug:<port>. We usually use port 9222 (unless another process is also using it.)

Once this is done, simply open localhost:<port> on your browser. You will see a list of links, each pertaining to a plugin/webpage URL source and labeled through their TITLE element, or if not present, their URL. Click the link pertaining to your plugin to begin debugging.

Debugging Your Source Properties Window

You can now also debug your source properties window. To do this, you would need to download the xsplitprops.html file here and move it to the same folder where XSplit.Core.exe is located.

After this, you would need to add the startup parameter: enablehtmlprop:2.

You will now then be able to see the XBC Source Properties_debugger item on the list of links together with your plugin/s. Click on this link and you can start debugging the source property window of plugin.

Note: The XBC Source Properties_debugger will allow you to debug the source properties of the source that was last opened. With that said, make sure that you open your plugin's source properties first before debugging.
