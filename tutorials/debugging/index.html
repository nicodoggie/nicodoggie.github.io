Debugging Your Plugins

Important Notice: Google Chrome removed support for remote debugging starting from Chrome 50. For developers who updated to the said version or higher, you may use the following alternatives:
  • load the remote debugging discovery page as an extension in XBC OR
  • download independent CEF builds (Chromium 49 and lower) from https://cefbuilds.com/and use them to load the remote debugging discovery page

XSplit Broadcaster allows you to debug your pages by exposing the DevTools. To do this, create a shortcut to XSplit.Core.exe, and right-click to open the properties window to add the startup parameter: remotedebug:<port>. We usually use port 9222 (unless another process is also using it.)

Once this is done, simply open localhost:<port> (the remote debugging discovery page) on your browser. You will see a list of links, each pertaining to a plugin/webpage URL source and labeled through their TITLE element, or if not present, their URL. Click the link pertaining to your plugin to begin debugging.

Important note: Debugging for the source properties window is not available at this time.
