Debugging Your Plugins

Important Notice: Google Chrome removed support for remote debugging starting from Chrome 50. For developers who updated to the said version or higher, you may use the following alternatives:
  • load the remote debugging discovery page as an extension in XBC; or
  • download an independent CEF build from https://cefbuilds.com/ and use them to load the remote debugging discovery page. (Click on a "Test App" link for any item that's Chromium 49 at most.)

XSplit Broadcaster allows you to debug your pages by exposing the DevTools. To do this, create a shortcut to XSplit.Core.exe, and right-click to open the properties window to add the startup parameter: remotedebug:<port>. The default is port 9222, but you can use your preferred port (just ensure that no other process is also using it.)

Once this is done, simply open http://localhost:<port> (the remote debugging discovery page) on your browser. You will see a list of links, each pertaining to a plugin/webpage URL and labeled through their <title> element, or if not present, their URL. Click the link pertaining to your plugin to begin debugging. Note that this is not just for sources, but for extensions as well!

Debugging Your Source Properties Window

Debugging the source properties window is a bit tricky. To do this, you would need to download the xsplitprops.html file here and move it to the same folder where XSplit.Core.exe is located. This is a special build for debugging.

After this, you would need to add the startup parameter to XSplit: enablehtmlprop:2.

You will now then be able to see the XBC Source Properties_debugger item on the remote debugging page, together with your plugins. Click on this link and you can start debugging the source property window you wrote.

Note: The XBC Source Properties_debugger will allow you to debug the source properties of the source that was last right-clicked. With that said, make sure that you open your plugin's source properties first before debugging XBC Source Properties_debugger.
