Start/Stop Broadcast

Introduction

XSplit now allows users the ability to Start or Stop a Broadcast through the framework and Pause or Unpause a Local Recording. This will be really useful for things such as Scheduled Streaming.

How to Start your Broadcast

Before one can start broadcasting, you should know your available Output Channels and which ones you want to start broadcasting to. 


// get necessary classes
var xjs = require('xjs');
var Output = xjs.Output;

xjs.ready()
  .then(Output.getOutputList)
  .then(function(outputs) {
    outputs.map(output => {
      output.getName()
      .then(function(name) {
        if(name.includes('Twitch')) {
          output.startBroadcast();
        }
      })
    })
  })

This will provide you an array of all the Output instance that's available/installed on your XSplit Broadcaster. You can then continue to select which instance you would like to start or stop broadcasting to.

Once you've selected the output instance you want to start broadcasting to. You can just immediately start/stop the broadcast from there. 


  output.startBroadcast();
  // or
  output.stopBroadcast();

Additionally, if you're only on a Local Recording, you can choose to call output.pauseLocalRecording() or output.unpauseLocalRecording() to pause/unpause the local recording.

⚠Current Limitation

Output plugins such as YouTube or Facebook that has pre streaming dialogs, would still need to be set-up manually. Any other output plugins would work immediately using the predefined configuration the user has saved initially.
