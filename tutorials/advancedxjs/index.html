Installing XJS - Advanced

Instead of creating your HTML and JS files from scratch as described in Quick Start, there are still other ways you can install xjs in your projects. You can select any of the following methods:

  • Scaffold a project with Yeoman
  • Dependency management with npm
  • Dependency management with Bower
  • Start from a zip file
  • Direct link to hosted cdn file

Scaffold a project with Yeoman

Dependencies

You'll need to have Node.js and npm installed first. If you have that installed already, great! If not, you should go to the official website, download and install it. Also, do make sure that you have the latest version of npm to avoid issues with other tools. Finally, you will need Git on your machine.

Installation

Let's get started first with installing yeoman and XSplit JS Framework generator through npm. 

npm install -g bower
npm install -g yo
npm install -g generator-xjs

That will allow you to generate XSplit JS Framework based projects through the command yo xjs.

To use our generator, you'll have to create your project folder, and then scaffold your project in there: 

mkdir YOUR_DIR
cd YOUR_DIR
yo xjs

You will be prompted with a number of options like project name, version, plugin type, and package name.

That should create the basic structure for your plugin, including a generic HTML file you can use to get started. The directory should now also include the file xjs.js, which contains the framework code. We also included browsersync for you to test your plugin on a browser first; all you have to do is execute gulp in the root folder of your project. This serves your plugin at http://localhost:9001, and watches your code for changes so it can automatically refresh your HTML.

Dependency management with npm

Usage of npm is straightforward.

Installation

Assuming npm is installed, just execute this in your command line: 

npm install xjs-framework

This would create a node_modules folder. The relevant file is node_modules/xjs-framework/dist/xjs.js, which you should include in your plugin's HTML file. All other details of the project structure are up to you.

This approach also opens up an alternative coding method. You can create your plugins using TypeScript or ES2015 if desired. To learn more, see our tutorials.

Dependency management with Bower

Our Yeoman Generator also makes use of Bower, but the generator simply executes it automatically.

Installation

To use bower, install it first, and then fetch xjs-framework through bower. 

npm install -g bower
bower install xjs-framework

This creates a bower_components folder. The relevant file is bower_components/xjs-framework/dist/xjs.js, which you should include in your plugin's HTML file. All other details of the project structure are up to you. See the following example for what a basic plugin might look like.

Example:

<html>
  <body>
  <script src="bower_components/xjs-framework/dist/xjs.js"></script>
  <script src="js/main.js"></script>
  </body>
</html>

Getting started from ZIP files

In case you need a very simple approach, you can just use this zip file. It contains a basic folder structure for a source plugin (but it can be used for extensions as well.) Please click the button below to download it.

In case you need to update the framework to the latest available version, you could always execute bower update on your CLI.

Getting started using our CDN hosted file

In case you need to create a plugin really quickly, all you have to do is add this line to your html file: 

<script src="https://cdn2.xsplit.com/xjs/download/xjs.js"></script>

Or use the minified version:

<script src="https://cdn2.xsplit.com/xjs/download/xjs.min.js"></script>

The problem with this method is that our servers cache static files, thus we cannot ensure that the file hosted is the latest version of the framework, especially if we just released a new version of the framework within the past 24 hours. We recommend exploring the other options instead of linking to our CDN if you are developing a plugin, and not merely testing out the capabilities of the framework.
